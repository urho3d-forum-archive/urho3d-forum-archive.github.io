inqubator:

CharacterDemo seems to have “patchInstructions” variable etc. for on-screen joystick; how can this be enabled?

You don’t have to use “patching” approach as the sample. You can define a complete on-screen joystick UI layout using the Editor app or hand-coded the XML manually. Since we have a lot of sample apps and most of them can share a basic same layout, our approach is to use this basic UI layout and then specialized it as needed using the XML patching mechanism for each sample. But to answer your question directly, there is no special option required to enable this XML patching mechanism. The patching mechanism is available for any XML resources and not just limited to UI layout XML file. See github.com/urho3d/Urho3D/pull/119 for more detail. You may have to look into the XMLFile::Patch() method to see how it works.
