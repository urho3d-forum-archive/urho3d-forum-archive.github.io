theron-library.com/

If I had the time/motivation/skillset, a cool toy engine could be made with this (or at least learning the concepts so it works well with your codebase), perhaps also implementing everything into job tasks with pure functions and AoS/SoA structs pumped through code written in ISPC. I think also decoupling the development pipeline and the runtime as another cool way to go, like just doing XML stuff in the development phase and then transforming this to a binary format which is just directly mapped to a struct. The tool pipeline can be just Qt+Python and as procedural as possible.
